# Gallery photos and Memory personal photos never overlap

A guest's Memory form allows two different photo uploads: a personal photo of themselves (or with Nirmal), and separate photos of Nirmal contributed to the Gallery. We considered merging these — any attached photo also flows into the Gallery — but decided to keep them strictly separate: personal photos stay inside the Memory card only; Gallery photos (from the form's dedicated step, or the low-key "+" button on the Gallery page itself) are the only ones that ever appear in the shared Gallery.

This avoids the Gallery filling with selfies unrelated to Nirmal's life, and keeps both the data model and the admin moderation surface simpler. Reversing this later would mean re-scoping every existing personal photo, so it's worth recording now rather than letting the two concepts blur.
